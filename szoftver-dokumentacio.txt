1. A SZOFTVER CÉLJA ÉS ÁTTEKINTÉSE
───────────────────────────────────────────────────────────────────

A Planova egy felhasználóközpontú naptár- és produktivitás-alkalmazás, amely
lehetővé teszi a felhasználók számára naptárak kezelését, események és feladatok
ütemezését, szokások követését, barátok kezelését, valamint naptárak megosztását
másokkal. Az alkalmazás elérhető reszponzív webes felületen és natív mobil
alkalmazásként egyaránt.

A szoftver célja, hogy egyetlen platformon egyesítse a személyes időkezelés és
produktivitás-követés legfontosabb eszközeit, miközben biztosítja a valós idejű
együttműködést és a több eszközön átívelő szinkronizációt.


2. ARCHITEKTÚRA ÉS TECHNOLÓGIAI HÁTTÉR
───────────────────────────────────────────────────────────────────

2.1. Általános felépítés

Az alkalmazás monorepo struktúrában épül fel, pnpm workspaces és Turborepo
segítségével. A projekt három fő alkalmazásból és hat megosztott csomagból áll.

2.2. Backend

  - Keretrendszer: .NET 8 (C#)
  - Adatbázis: PostgreSQL, Entity Framework Core ORM-mel
  - Hitelesítés: JWT Bearer tokenek (access + refresh token)
  - Valós idejű kommunikáció: SignalR hub
  - API dokumentáció: Swagger / OpenAPI

2.3. Webes frontend

  - Keretrendszer: Next.js (App Router)
  - UI könyvtár: Tamagui
  - Állapotkezelés: Zustand (store-ok), React Query (szerver adat)
  - Típusosság: TypeScript, OpenAPI-ból generált típusok

2.4. Mobil alkalmazás

  - Keretrendszer: Expo / React Native
  - Navigáció: expo-router (fájl-alapú routing)
  - UI könyvtár: Tamagui (közös a webbel)
  - Token tárolás: expo-secure-store

2.5. Megosztott csomagok

  - @repo/api – Axios-alapú API kliens, token-frissítő interceptorokkal
  - @repo/types – Swagger/OpenAPI-ból generált TypeScript típusok
  - @repo/features – Közös UI komponensek (AuthCard, Calendar, CalendarEntry stb.)
  - @repo/hooks – Zustand store-ok és React Query hook-ok
  - @repo/ui – Stílusos UI primitívek (gombok, inputok, select elemek)
  - @repo/utils – Naptárrács-generálás, dátum segédfüggvények, szín segédletek
  - @repo/config – Tamagui konfiguráció


3. FUNCIÓK RÉSZLETES LEÍRÁSA
───────────────────────────────────────────────────────────────────

3.1. Regisztráció és hitelesítés

  - E-mail/jelszó alapú regisztráció erős jelszó-szabályokkal
    (min. 8 karakter, kis- és nagybetű, szám, speciális karakter)
  - Regisztrációkor automatikusan létrejön a felhasználói profil
    és egy alapértelmezett naptár
  - JWT access + refresh token rendszer
  - Webes felületen: HttpOnly cookie-k (SameSite=Strict)
  - Mobil alkalmazásban: expo-secure-store biztonságos tárolás
  - Automatikus token-frissítés: 401-es válasz esetén az API kliens
    automatikusan frissíti a tokent és újrapróbálja a kérést

3.2. Profilkezelés

  - Felhasználónév, avatar, vezetéknév, keresztnév, születési dátum kezelése
  - Adatvédelmi beállítás: publikus vagy privát profil
  - E-mail cím megjelenítése és a fiók létrehozási dátuma

3.3. Naptárkezelés

  - Több naptár létrehozása, szerkesztése és törlése
  - Minden naptárhoz egyedi név és szín rendelhető (12 szín paletta)
  - Naptárak megjelenítésének ki/be kapcsolása az oldalsávban
  - A naptár törlése automatikusan törli az összes hozzá tartozó bejegyzést
    és megosztást

3.4. Naptárbejegyzések

  - Bejegyzés kategóriák: Nincs, Feladat, Esemény, Születésnap, Évforduló
  - Minden bejegyzésnek van neve, opcionális leírása, kezdő- és záró dátuma,
    helyszíne, értesítési ideje, színe és egész napos jelzője
  - Feladatoknál teljesítettségi állapot követése
  - Bejegyzések szín alapján vizuálisan megkülönböztethetők
  - Kontextus menü: jobb klikkel gyorsan létrehozható új esemény
    egy adott napon

3.5. Naptárnézetek

  Az alkalmazás négy különböző nézetet támogat:

  - Havi nézet – Teljes hónap rács hétszámokkal, egérgörgővel navigálható
  - Többhetes nézet – 4 hetes áttekintés
  - Heti nézet – Egyetlen hét óránkénti időosztással (24 órás görgethető nézet)
  - Napi nézet – Egyetlen nap óránkénti időosztással

  Minden nézetben megjelennek a hétszámok és a bejegyzések szín-kódolva.
  Heti és napi nézetben az átfedő események egymás mellett jelennek meg.
  Ha egy cellában nem fér el minden bejegyzés, „+X további" jelzés mutatja
  a rejtett elemek számát.

3.6. Megosztott naptárak

  - Bármely naptár megosztható más felhasználókkal
  - Két jogosultsági szint: Szerkesztő és Megtekintő
  - A megosztott naptárak a saját naptárak mellett jelennek meg
  - A jogosultsági szint később módosítható, a megosztás visszavonható

3.7. Barátrendszer

  - Barátkérés küldése más felhasználóknak
  - Barátkérés elfogadása vagy elutasítása
  - Barátok listázása állapot szerint (Elfogadott, Függőben, Elutasított)
  - Barátság törlése
  - Barátkérés küldésekor valós idejű értesítés SignalR-en keresztül

3.8. Valós idejű értesítések

  - SignalR hub a /notifications végponton
  - Automatikus újracsatlakozás kapcsolat megszakadása esetén
  - Felhasználó-azonosítás a JWT token ProfileId claim-je alapján
  - Webes felületen a cookie-k automatikusan továbbítódnak
  - Mobil alkalmazásban az access token query string-ként kerül átadásra
  - Az értesítések Zustand store-ban tárolódnak

3.9. Szokáskövetés

  - Szokások létrehozása, szerkesztése és törlése
  - Szokás kategóriák: Testmozgás, Olvasás, Tudatosság
  - Mértékegységek: Távolság, Darabszám, Idő
  - Célérték megadása és napi naplózás
  - Az egyes napokra megadható, hogy melyik napokon aktív a szokás
  - Szokás törlése automatikusan törli az összes naplóbejegyzést

3.10. Esemény résztvevők (tervezett funkció)

  - Az adatmodell már tartalmazza az EventContributor entitást
  - Profil és naptárbejegyzés összerendelése Függőben/Elfogadott/Elutasított
    státusszal
  - A kontroller implementálása még folyamatban van


4. ADATMODELL
───────────────────────────────────────────────────────────────────

4.1. Entitások

  - User – Felhasználói fiók (e-mail, jelszó hash, refresh token)
  - Profile – Felhasználói profil (felhasználónév, avatar, adatvédelem)
  - Calendar – Naptár (név, szín, profilhoz kötve)
  - CalendarEntry – Naptárbejegyzés (kategória, név, leírás, dátumok, szín)
  - Habit – Szokás (név, kategória, mértékegység, cél, napok)
  - HabitLog – Szokás napló (dátum, érték, teljesítettség)
  - Friend – Barátság (két profil között, státusz)
  - SharedCalendar – Megosztott naptár (profil + naptár + jogosultság)
  - EventContributor – Esemény résztvevő (profil + bejegyzés + státusz)

4.2. Kapcsolatok

  - Egy felhasználóhoz egy profil tartozik
  - Egy profilhoz több naptár, szokás, barátság és megosztott naptár tartozhat
  - Egy naptárhoz több bejegyzés és megosztás tartozhat
  - Egy szokáshoz több naplóbejegyzés tartozhat
  - A barátság két profil közötti kétirányú kapcsolat


5. FELHASZNÁLÓI FELÜLET
───────────────────────────────────────────────────────────────────

5.1. Webes útvonalak

  - /sign-in – Bejelentkezés / Regisztráció (AuthCard komponens)
  - /calendar – Fő naptár nézet (védett útvonal)
  - /profile – Profil szerkesztés

5.2. Mobil képernyők

  - Vendég útvonalak: Kezdőlap, Bejelentkezés
  - Védett útvonalak: Naptár

5.3. Navigációs sáv (Navbar)

  - „Planova" logó
  - Oldalsáv ki/be kapcsoló gomb
  - Avatár és beállítások felugró ablak:
    - Téma váltás (világos/sötét)
    - Hét kezdőnapja (hétfő/vasárnap)
    - Óra formátum (12 órás / 24 órás)
  - Kijelentkezés gomb

5.4. Oldalsáv (Sidebar)

  - Mini naptár dátum navigációhoz
  - Naptárlista jelölőnégyzetekkel (naptárak megjelenítése/elrejtése)
  - Naptáranként opciók (szerkesztés, törlés)
  - Új naptár létrehozása gomb

5.5. Reszponzív megjelenés

  - Tablet töréspontnál (<1024px): automatikusan váltás kisebb nézetekre
  - Mobil töréspontnál (<768px): további nézetméret csökkentés
  - Automatikusan eltűnő oldalsáv kisebb kijelzőkön


6. BIZTONSÁG
───────────────────────────────────────────────────────────────────

  - Jelszavak hash-elve tárolódnak az adatbázisban
  - JWT tokenek rövid élettartammal, refresh token rotáció
  - HttpOnly cookie-k a webes felületen (XSS védelem)
  - SameSite=Strict cookie beállítás (CSRF védelem)
  - expo-secure-store a mobil alkalmazásban (titkosított tárolás)
  - Minden védett végpont [Authorize] attribútummal ellátva
  - Erős jelszó-szabályok a regisztrációnál


7. KERESZTPLATFORMOS KÓDMEGOSZTÁS
───────────────────────────────────────────────────────────────────

A monorepo struktúra lehetővé teszi a webes és mobil alkalmazás között
jelentős kódbázis megosztását:

  - Hitelesítési UI (AuthCard, SignInForm, SignUpForm)
  - Naptár komponensek (Calendar, CalendarCell, CalendarEntry)
  - Védett útvonal logika (ProtectedRoute)
  - Összes Zustand store és React Query hook
  - Teljes API kliens réteg
  - Segédfüggvények és típusdefiníciók

Ez biztosítja az egységes működést mindkét platformon, miközben
minimalizálja a kód duplikációt.


8. TÍPUSGENERÁLÁS ÉS API SZINKRONIZÁCIÓ
───────────────────────────────────────────────────────────────────

A backend Swagger/OpenAPI specifikációt generál, amelyből az
openapi-typescript csomag automatikusan létrehozza a TypeScript
típusdefiníciókat. Ez biztosítja, hogy a frontend típusai mindig
szinkronban legyenek a backend API-val, és a típushibák fordítási
időben kiderüljenek.
